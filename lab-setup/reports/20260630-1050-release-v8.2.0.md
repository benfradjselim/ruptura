## Bugs fixed

### BUG 2 — Frozen signals (all zero despite 18.4 M ingested metrics)

**Root cause**: OTel kubeletstats receiver emits metrics with dot-namespaced names
(`k8s.pod.cpu.node.utilization`, `k8s.pod.memory.working_set.utilization`). The
analyzer's signal processor expected `cpu_percent` / `memory_percent`. No name matched
→ all stress/fatigue/contagion signals stayed at zero.

**Fix** (`cmd/ruptura/main.go`): Added `mapToAnalyzerSignals()` translation shim in the
ticker loop. It copies raw OTel names into the map and then injects `cpu_percent` and
`memory_percent` by walking a priority list of kubeletstats CPU and memory metric names.
Also updated `pickPrimaryMetric()` to prefer translated names first.

### BUG 3 — Fleet charts / history / forecast empty

**Root cause**: Fleet API response schema uses `host` (e.g. `demo-cpu/host/cpu-hog-…`)
as the workload key. `Fleet.svelte` was written to read `w.key`, `w.name`,
`w.workload.namespace`/`w.workload.name` — none of which exist in the actual payload.
Forecast API was also called as `/forecast/{metric}/{ns}/{workload}` (three segments)
but the handler reads `vars["host"]` (one segment), returning 422 for every call.

**Fix** (`ui/src/pages/Fleet.svelte`, `ui/src/lib/api.js`): Introduced `hostKey(w)` and
`parseHost(w)` helpers; replaced all `w.key`/`w.name` references with `hostKey(w)`.
Changed forecast API call to two-segment form `/forecast/{metric}/{host}` and updated
`api.js` accordingly. Fixed signals tab to fall back to top-level KPI fields when
`w.signals` is missing.

### BUG 4 — Health score displayed as 8000 %

**Root cause**: `analyzer.go` line 532 converted `healthScore` (computed in `[0,1]`)
to `[0,100]` before storing in the KPI snapshot. `Fleet.svelte` then multiplied by 100
again when rendering.

**Fix** (`internal/analyzer/analyzer.go`): Removed the `*100` from the KPI snapshot
`Value` assignment (line 532). Removed corresponding `*100` in `ForecastHealthScore`
(stable case, line 884) and `project()` return (line 892). Adjusted rounding precision
from `*10/10` to `*1000/1000`. `Fleet.svelte` `(snap.health_score * 100).toFixed(0)`
is now correct.

### BUG 6 — Stale workload count (60 entries, dead pods never evicted)

**Root cause**: `AllSnapshots()` in `storage.go` returned every snapshot ever written
(deduplicated by canonical key) with no time-based eviction. Dead pods from previous
deployments persisted indefinitely.

**Fix** (`internal/storage/storage.go`): Added `snapshotStaleTTL = 5 * time.Minute`
constant. `AllSnapshots()` now skips any snapshot whose `Timestamp` is older than the
cutoff before deduplication. Count dropped from 60 → 35 active workloads.

### BUG 8 — Catalog card names and metric strings clipped or unstyled

**Root cause**: `.cc-name`, `.cc-desc`, `.p-title`, `.p-metric` CSS rules had no
overflow handling — long template names overflowed their card bounds.

**Fix** (`ruptura-autopilot/ui/src/pages/Catalog.svelte`): Added
`overflow: hidden; text-overflow: ellipsis; white-space: nowrap` to `.cc-name`,
`.p-title`, `.p-metric`. Added `-webkit-line-clamp: 3` multi-line truncation to
`.cc-desc`.

## Deploy steps

```
# 1. Community engine — tag triggers release.yml (Docker push)
git tag v8.2.0 && git push origin v8.2.0
# → ghcr.io/benfradjselim/ruptura:v8.2.0 pushed ✓

# 2. Autopilot proxy — tag triggers release.yml
cd /root/ruptura-autopilot
git tag v7.2.4 && git push origin v7.2.4
# → ghcr.io/benfradjselim/ruptura-autopilot:v7.2.4 pushed ✓

# 3. Helm upgrade (ruptura-autopilot namespace)
helm upgrade ruptura /root/ruptura-autopilot/helm/autopilot \
  --namespace ruptura-autopilot \
  --set engine.image.tag=v8.2.0 \
  --set proxy.image.tag=v7.2.4 \
  --reuse-values \
  --wait --timeout=180s
# → Release "ruptura" has been upgraded. REVISION: 3

# 4. Force pod replacement
kubectl rollout restart deployment/ruptura-ruptura-autopilot-engine \
  deployment/ruptura-ruptura-autopilot-proxy -n ruptura-autopilot
# → both deployments successfully rolled out
```

## Notes

- `mapToAnalyzerSignals()` is intentionally placed in `cmd/ruptura/main.go` rather than
  the ingest layer so the raw OTel names remain in storage (metric history, rollups). Only
  the analyzer sees the translated names.
- Image verification used GHCR registry API (Bearer token via `ghcr.io/token` endpoint)
  rather than `docker manifest inspect` (no docker CLI) or GitHub API (rate-limited).
- The community UI image (`ruptura-ui:ui-v7.0.26`) was not rebuilt — BUG 8 lives in the
  autopilot proxy's embedded Svelte UI, not in the community Nginx+Svelte pod.
- `main_test.go` version constant updated to `8.2.0` before tag push to keep
  `TestVersion_Constant` green.
