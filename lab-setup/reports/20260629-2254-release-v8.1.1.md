## Deploy steps

```
helm upgrade ruptura /root/ruptura/helm \
  --namespace ruptura \
  --set image.tag=v8.1.1 \
  --reuse-values \
  --wait --timeout=120s
# → Release "ruptura" has been upgraded. REVISION: 10

kubectl rollout restart deployment/ruptura deployment/ruptura-ui -n ruptura
# → both deployments successfully rolled out
```

Rollout restart was needed because both images were already tagged `v8.1.1` from a prior build; Kubernetes would not have pulled the new digest without it.
