# Autopilot v8.2.0 — Final Bug Report (2026-06-30)

## Verification: two open items from deploy review

### "fleet/infra: check failed" — root cause

The failure was a **wrong path** in the verification script. The script hit
`/api/v2/fleet/infra` (which returns 404) instead of the correct endpoint:

```
GET /api/v2/infra/groups
```

Verified:
```
curl -sk http://212.2.241.251:31480/api/v2/infra/groups \
  -H "Authorization: Bearer ruptura-lab-key"
→ HTTP 200  {"groups":[]}
```

The endpoint is live on the engine (port 31480) and also reachable through the proxy
(port 31482) — both return `{"groups":[]}`. The empty array is expected: the infra
collector DaemonSet from the v8.0 platform extension is not yet deployed in this
environment (BUG 5 below).

### Proxy route allowlist — does it block /api/v2/infra/*?

**No.** Full analysis of `internal/proxy/proxy.go` and `internal/proxy/rbac.go`:

- `rbac.go` (`Allowed()`) is **role-based only** — admin=full, operator=GET+approve/reject,
  viewer=GET. It contains no path blocklist.
- The master key (`ruptura-lab-key`) bypasses RBAC entirely via `proxy.go:247`:
  `if s.masterKey != "" && key == s.masterKey { s.rp.ServeHTTP(rec, r); return }`.
  The request goes directly to the engine without any path check.
- All paths not explicitly registered in the mux fall to
  `s.mux.HandleFunc("/", s.tenantProxy)` which proxies them to the engine.
- `/api/v2/infra/*` is not in any allowlist or blocklist.

The proxy at v7.2.4 is **transparent** to infra endpoints. No proxy update is required
for `/api/v2/infra/*` to be reachable from the browser.

### BUG 2 — Numbers frozen (KPI signals never update)

**Status: FIXED ✓**

**Root cause:** The OTel kubeletstats receiver emits metric names with dots
(`k8s.pod.cpu.node.utilization`). The analyzer expects `cpu_percent` / `memory_percent`.
No names matched → all signals computed as zero and never updated.

**Fix (`cmd/ruptura/main.go`):** Added `mapToAnalyzerSignals()` in the ticker loop.
It translates kubeletstats CPU and memory metric names to `cpu_percent`/`memory_percent`
by priority list, preserving the original names in storage.

Verified (post-deploy):
```
Non-zero stress: 26/37 workloads
stress sample: 0.4895, fatigue sample: 0.6164
```

---

### BUG 3 — No charts / no history / no forecast curve

**Status: FIXED ✓**

**Root cause:** Fleet API returns `{host: "ns/kind/name", health_score: ..., stress: ...}`.
`Fleet.svelte` was reading `w.key`, `w.name`, `w.workload.namespace`, `w.workload.name`
— none present in the actual response. Forecast was called as
`/forecast/{metric}/{ns}/{workload}` (three-segment) but the handler reads `vars["host"]`
(one-segment), returning 422 for every request.

**Fix (`ui/src/pages/Fleet.svelte`, `ui/src/lib/api.js`):** Added `hostKey(w)` and
`parseHost(w)` helpers; replaced all `w.key`/`w.name` references. Changed forecast to
two-segment `/forecast/{metric}/{host}` and updated `api.js`.

Verified: Fleet page renders signal history and forecast tabs without errors.

---

### BUG 4 — health_score = 8000%, signals exceed 100%

**Status: FIXED ✓**

**Root cause:** `analyzer.go` line 532 multiplied the already-computed `[0,1]`
`healthScore` by 100 before writing it to the KPI snapshot. `Fleet.svelte` then
multiplied by 100 again when rendering → 10000x overscale.

**Fix (`internal/analyzer/analyzer.go`):** Removed `*100` from snapshot `Value` (line
532), `ForecastHealthScore` stable case (line 884), and `project()` return (line 892).
Adjusted rounding from `*10/10` to `*1000/1000`.

Verified:
```
health_score range: [0.4008, 0.7250]
Hosts with health_score > 1.0: 0
```

### BUG 5 — No v8 infra features present

**Status: PARTIAL ✓ / Deployment gap remaining**

**Root cause:** The autopilot engine was running the community image at v7.1.0, which
predated all v8.x work. The infra endpoints simply did not exist.

**What's fixed:** The engine is now at v8.2.0 which includes the dual-axis infra
collector, CGPM, and all v8 API endpoints. `/api/v2/infra/groups` returns HTTP 200
through both the engine and proxy.

**Remaining gap:** `{"groups":[]}` — the infra collector DaemonSet from the v8.0
platform extension (`docs/v8.0.0/RUPTURA-PLATFORM-EXTENSION.md`) is not deployed in
this environment. The collector watches Kubernetes node/namespace/pod objects and
populates the infra registry. Without it, the groups array stays empty. This is a
deployment task, not a code bug.

Infra endpoint verified reachable:
```
Engine (31480): GET /api/v2/infra/groups → 200 {"groups":[]}
Proxy  (31482): GET /api/v2/infra/groups → 200 {"groups":[]}
```

### BUG 6 — Workload count stuck (dead pods never evicted)

**Status: FIXED ✓**

**Root cause:** `AllSnapshots()` in `storage.go` returned every snapshot ever written,
with no time-based eviction. Dead pods accumulated indefinitely.

**Fix (`internal/storage/storage.go`):** Added `snapshotStaleTTL = 5 * time.Minute`.
`AllSnapshots()` skips snapshots older than the cutoff before deduplication.

Verified:
```
Pre-fix:  60 workloads (dead pods accumulated over 3 days)
Post-fix: 37 workloads (only live pods with fresh snapshots)
```

---

### BUG 7 — Topology page empty

**Status: SKIPPED — not fixable without trace data**

**Investigation:** The topology API (`/api/v2/topology`) returns 60 nodes / 0 edges.
Edges require OTel trace spans (`traces: 0` in the response) — no trace source is
configured in the OTel Collector ConfigMap, only kubeletstats and prometheus receivers.
Without trace data, the edge inference engine produces no graph.

Additionally, there is no `Topology.svelte` page in the active Svelte UI at
`workdir/ui/src/pages/`. Fixing this would require:
1. Adding a trace receiver to the OTel Collector DaemonSet, **and**
2. Creating a new `Topology.svelte` page with a graph renderer.

Deferred to a future release.

---

### BUG 8 — Dashboard catalog: template names truncated / preview invisible

**Status: FIXED ✓**

**Root cause:** `.cc-name`, `.cc-desc`, `.p-title`, `.p-metric` CSS rules in
`ruptura-autopilot/ui/src/pages/Catalog.svelte` had no overflow handling.

**Fix:** Added `overflow: hidden; text-overflow: ellipsis; white-space: nowrap` to
`.cc-name`, `.p-title`, `.p-metric`. Added `-webkit-line-clamp: 3` multi-line clamping
to `.cc-desc`.

Part of autopilot proxy rebuild v7.2.4.

---

### BUG 9 — Dashboard builder limited panel types

**Status: DEFERRED**

Feature gap, not a bug. The builder's signal type list is hardcoded to ~10 options.
Expanding it requires either a dynamic signal catalog endpoint or a larger hardcoded
list with the newly-available v8 infra signals. Lowest priority in the original brief
— not addressed in this release.
