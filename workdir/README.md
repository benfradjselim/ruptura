# OHE — Observability Holistic Engine

> **Prevention is better than cure.** OHE is a self-hosted, Kubernetes-native observability platform with ML-powered predictive alerting. It monitors your infrastructure today and warns you about tomorrow's incidents.

---

## What makes OHE different

Most observability tools tell you what broke. OHE tells you what **will** break — hours before it does.

| Capability | What it does |
|-----------|-------------|
| **Holistic KPIs** | 10 composite signals: stress, fatigue, mood, pressure, humidity, contagion, resilience, entropy, velocity, health_score |
| **ML Forecasting** | Exponential smoothing + trend extrapolation. Predicts CPU/memory exhaustion hours ahead |
| **SLO Tracking** | Define targets, track error budget burn rate, alert on budget exhaustion |
| **Long-term Retention** | 3-tier storage: raw 7d → 5m rollups 35d → 1h rollups 400d. Never lose history again |
| **Multi-tenancy** | Org workspaces with RBAC — admin/operator/viewer roles, user→org assignment |
| **Zero-dep binary** | Single binary + BadgerDB. No Postgres, no Kafka, no Elasticsearch |
| **Protocol compatibility** | Prometheus, OTLP, Loki, Elasticsearch Bulk, Datadog, DogStatsD |
| **Predictive dashboards** | Next 1h / 6h / 24h time range on any dashboard panel |

## Quickstart (k3s / Kubernetes)

```bash
kubectl apply -f deploy/pvc.yaml
kubectl apply -f deploy/secrets.yaml
kubectl apply -f deploy/configmap.yaml
kubectl apply -f deploy/rbac.yaml
kubectl apply -f deploy/central-deployment.yaml
kubectl apply -f deploy/agent-daemonset.yaml
```

Default login on first boot — printed to pod logs:
```
Username: admin
Password: <auto-generated>   # kubectl logs -n ohe-system deploy/ohe-central
```

Override: `OHE_ADMIN_PASSWORD=yourpassword`

## v4.4.0 — What's New

### SLO / Error Budget Engine
- **SLO CRUD** — `GET/POST /api/v1/slos`, `GET/PUT/DELETE /api/v1/slos/{id}`
- **Live status** — `GET /api/v1/slos/{id}/status` and `GET /api/v1/slos/status` (all)
- Computes: compliance %, error budget remaining %, burn rate, remaining minutes, state
- States: `healthy` | `at_risk` | `breached` | `no_data`
- **SLO widget** (`type: "slo"`) — detail view with budget bar + burn rate
- **SLOs page** — full management UI under Respond section

### Widget Resize
- ◀ ▶ ▲ ▼ resize buttons in dashboard edit mode
- Widgets store `w` (1–4 columns) × `h` (1–4 rows) on a 4-column CSS grid
- Size badge shown in edit cards; applied to view with `grid-column/row: span N`

### Gauge Widget — Full Redesign
- 300° arc sweep, responsive `viewBox="0 0 200 160"`
- Color: green/yellow/red based on value; inverted for stress/fatigue (lower = better)
- Drop-shadow glow, tick marks, pct label, min/max axis labels
- Fills container width — no more fixed 130px square

### 6 New Built-in Templates (20 total)

## v4.3.0 — What's New

### Long-term Retention + Downsampling
| Tier | Key prefix | Retention | Resolution |
|------|-----------|-----------|------------|
| Raw | `m:` / `k:` | 7 days | as-collected |
| 5-min rollup | `r5:` / `kr5:` | 35 days | 5-min avg |
| 1-hour rollup | `r1h:` / `kr1h:` | 400 days | 1-hour avg |

- Compaction goroutine runs every 30 min — raw→5m after 2h, 5m→1h after 12h
- `GET /api/v1/metrics/{name}/range` — auto-selects tier by window
- `GET /api/v1/retention/stats` — live data counts per tier
- `POST /api/v1/retention/compact` — on-demand compaction (operator+)

### Full RBAC
- `PUT /api/v1/auth/users/{id}/org` — assign user to org
- `GET /api/v1/orgs/{id}/members` — list org members
- `POST /api/v1/orgs/{id}/members` — invite user into org
- JWT claims carry `org_id`; propagated as request context
