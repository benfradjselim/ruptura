# OHE — Observability Holistic Engine

> **Prevention is better than cure.** OHE is a self-hosted, Kubernetes-native observability platform with ML-powered predictive alerting. It monitors your infrastructure today and warns you about tomorrow's incidents.

---

## What makes OHE different

Most observability tools tell you what broke. OHE tells you what **will** break.

- **Holistic KPIs** — stress, fatigue, pressure, resilience, entropy, velocity, health_score. System-level signals, not just raw metrics.
- **ML forecast engine** — exponential smoothing + trend extrapolation. Predicts CPU/memory exhaustion hours ahead.
- **Predictive dashboards** — "Next 1h / Next 6h / Next 24h" time range mode on any dashboard.
- **Zero infrastructure deps** — single binary, BadgerDB embedded storage. No Postgres, no Kafka, no Elasticsearch required to start.
- **Prometheus-compatible** — exposes `/metrics` in Prometheus exposition format. Accepts StatsD/DogStatsD on UDP :8125 and OTLP HTTP.

## Quickstart (k3s / Kubernetes)

```bash
# 1. Create namespace and apply storage
kubectl apply -f deploy/pvc.yaml

# 2. Apply secrets, config, RBAC
kubectl apply -f deploy/secrets.yaml
kubectl apply -f deploy/configmap.yaml
kubectl apply -f deploy/rbac.yaml

# 3. Deploy central + agent
kubectl apply -f deploy/central-deployment.yaml
kubectl apply -f deploy/agent-daemonset.yaml

# 4. (Optional) Deploy Prometheus in monitoring namespace
kubectl apply -f deploy/prometheus.yaml
```

Default login after first boot — credentials printed to pod logs:
```
Username: admin
Password: <auto-generated>
# See: kubectl logs -n ohe-system deploy/ohe-central
```

Override with env var: `OHE_ADMIN_PASSWORD=yourpassword`

## v4.1.0 — What's New

### Alert Delivery — webhook & Slack (fully wired)
Notification channels are now end-to-end functional:
- `POST /api/v1/notifications` — register a Slack or webhook channel
- Alerts automatically POST to all enabled channels when triggered
- Severity filtering: channel can receive only `critical`, `warning`, etc.
- Custom headers support (PagerDuty routing keys, Bearer tokens)
- `POST /api/v1/notifications/{id}/test` — send a live test payload

### Kubernetes Agent (fixed)
- Agent DaemonSet ships real host metrics (CPU, memory, disk, network) to central
- `OHE_CENTRAL_URL` env var fallback for when cluster DNS is unavailable

### Datasource SSRF hardening
- `*.svc.cluster.local` and `*.svc` hostnames allowed — safe in-cluster services
- `OHE_TRUSTED_DATASOURCE_HOSTS` env var for explicit ClusterIP allowlist

### Prediction chart fix
- Forecast dashboards show future timestamps on X-axis (Next 1h → labels from now → +1h)
- New `buildForecastTicks()` in svgChart.js aligns tick labels to forecast window

## API Reference

```
GET  /api/v1/health/live              liveness probe
GET  /api/v1/health/ready             readiness probe
GET  /api/v1/metrics                  current metric snapshot
GET  /api/v1/metrics/{name}           time-series range (?from=&to=&host=)
GET  /api/v1/kpis                     holistic KPI snapshot
GET  /api/v1/predict                  ML forecast (?host=&horizon=120)
GET  /api/v1/alerts                   alert list (?active=true)
GET  /api/v1/dashboards               dashboard list
POST /api/v1/templates/{id}/apply     apply a dashboard template
POST /api/v1/datasources              register external datasource
POST /api/v1/datasources/{id}/test    test datasource connectivity
POST /api/v1/notifications            register notification channel
POST /api/v1/notifications/{id}/test  fire test payload to channel
GET  /metrics                         Prometheus exposition format
```
